# Findings Report

## Objective

Develop a controlled phishing simulation laboratory using Gophish and Mailpit to understand the lifecycle of phishing campaigns from a defensive and awareness perspective.

---

## Scope

The laboratory was conducted in an isolated environment without involving real users, credentials, or production systems.

Technologies used:

* Kali Linux
* Gophish
* Mailpit
* Git
* GitHub

---

## Methodology

The following stages were performed:

1. Installation and configuration of Gophish.
2. Configuration of Mailpit as SMTP service.
3. Creation of email templates.
4. Creation of a simulated landing page.
5. Creation of a phishing campaign.
6. Monitoring campaign metrics.
7. Documentation of findings.

---

## Results

The laboratory successfully demonstrated:

* Email template creation.
* SMTP profile configuration.
* Landing page deployment.
* Campaign execution.
* Collection of user interaction metrics.

Observed metrics included:

* Email opens.
* Link clicks.
* Form submissions.
* Campaign status tracking.

---

## Security Awareness Perspective

The exercise highlights how phishing attacks can be structured and executed, emphasizing the importance of:

* Security awareness training.
* Email verification procedures.
* Multi-factor authentication.
* Suspicious link detection.
* User education programs.

---

## Lessons Learned

Key lessons obtained from this laboratory:

* Understanding phishing infrastructure.
* Building realistic phishing simulations.
* Monitoring campaign effectiveness.
* Documenting security exercises.
* Applying ethical and legal considerations during testing.

---

## Ethical Statement

This laboratory was performed exclusively for educational, defensive, and cybersecurity awareness purposes in a controlled environment.
